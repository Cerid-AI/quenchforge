# Code of Conduct

This project adopts the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as its Code of Conduct. The canonical text is authoritative; this file exists so contributors can find it from the repository root.

## TL;DR

- Be respectful. Disagree about code, not about people.
- Hardware-platform discussions can get tribal (Intel vs. ARM, AMD vs. NVIDIA, Mac vs. PC). Keep it technical.
- Triage reports require a `quenchforge doctor` paste — that's about reproducibility, not gatekeeping, and applies equally to maintainers.
- Assume good faith from contributors on hardware you don't own; assume the same in return.

## Scope

Applies to every interaction in this project's surfaces: GitHub Issues and Discussions, pull requests, code review, commit messages, release notes, the project chat (if and when one exists), and any in-person or virtual event held in the project's name.

## Reporting

Email `conduct@cerid-ai.com`. Reports are read by the project maintainers and handled per the Covenant's [enforcement guidelines](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines). If a report concerns a maintainer, escalate to `security@cerid-ai.com` so it lands with someone outside the maintainer group.

Reports stay confidential to the people handling them. Retaliation against reporters is itself a violation.
